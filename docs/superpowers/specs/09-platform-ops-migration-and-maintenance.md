# 平台交付、迁移与远程维护

## 环境分层

- `dev`
- `staging`
- `prod`

不同环境的数据库、对象存储、外部服务配置必须分离。

## 迁移原则

- 应用与配置分离
- 资产存对象存储
- schema 全部走 migration
- 模板、知识、prompt、规则支持导入导出
- 服务地址全部配置化

## 备份与恢复

必须覆盖：

- Postgres
- 对象存储
- 模板与知识资产
- 审计日志

## 远程维护工作流

- Git 仓库为唯一代码真源
- 远程开发机 / 服务器为维护环境
- 通过 SSH / VPN / 堡垒机访问
- 变更先入仓库，再测试，再发布
- 不在生产机上直接开发

## Codex 角色

Codex 作为维护助手参与：

- 读代码
- 写代码
- review
- 验证
- 文档维护

但不应成为运行时依赖。
